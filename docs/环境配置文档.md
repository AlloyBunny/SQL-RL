# SQL-RL 环境配置文档

## 1. 运行机器

本项目可能在两类机器上访问：

- MacBook：用于阅读代码、写文档、准备数据脚本、做轻量测试。
- GPU 服务器：用于真实训练和大规模推理。

GPU 服务器是搭载 `4 * H100` 的 CCI 容器。

CCI 容器的 SSH 连接时长只有 4 小时，所以任何需要长时间运行的任务都不能直接挂在前台终端里跑。

训练、批量推理、长时间评估必须使用 `nohup` 或类似方式后台运行，并记录 log。

示例：

```bash
nohup bash scripts/train_grpo.sh > logs/train_grpo_$(date +%Y%m%d_%H%M%S).log 2>&1 &
```

训练过程最好接入 SwanLab，用于观察 loss、reward、accuracy、invalid rate、timeout rate 等指标。

## 2. 代码管理

本项目需要使用 git / GitHub 管理代码。

基本要求：

- 初始化 git 仓库。
- 设置 GitHub remote。
- 关键改动及时 commit。
- 文档、配置、核心代码都纳入版本管理。
- 数据集、模型权重、训练输出、日志大文件不直接提交。

建议 `.gitignore` 包含：

```text
.venv/
outputs/
logs/
checkpoints/
data/raw/
data/cache/
*.log
*.safetensors
*.bin
```

## 3. CCI 容器环境保留问题

CCI 容器每次重新 SSH 连接后，之前手动配置的环境可能会丢失。

解决方案有两个：

1. 使用 uv 管理 Python 虚拟环境。
2. 保存容器镜像。

第一阶段优先使用 uv 管理 Python 环境。原因是项目第一版尽量避免依赖复杂 C/CUDA 编译链，优先使用已有 wheel 和训练框架提供的安装方式。

如果后续必须安装 vLLM、flash-attn、deepspeed、特定 CUDA 编译依赖，且反复重装成本过高，再考虑保存容器镜像。

## 4. uv 使用策略

计划在项目根目录创建 uv 环境：

```bash
uv venv --python 3.10
source .venv/bin/activate
uv pip install -U pip
```

后续安装 Python 包优先使用：

```bash
uv pip install <package>
```

注意：uv 可以管理 Python 包和虚拟环境，但不能替代 GPU 驱动、系统 CUDA 驱动、NCCL、容器底层镜像等系统级依赖。

如果只做数据处理、SQLite reward、evaluation，MacBook 和 CPU 环境都可以运行一部分逻辑。

如果做模型训练，仍然需要 GPU 服务器上的 PyTorch CUDA 环境。

## 6. 推荐环境路线

第一步：MacBook 只做文档、项目结构、轻量 Python 脚本，不装 GPU 训练环境。

第二步：GPU 服务器上用 uv 创建项目环境。

第三步：先安装 ms-swift 路线需要的最小依赖。

第四步：先跑通数据处理、prompt、SQLite reward、evaluation。

第五步：再启动 SFT。

第六步：最后启动 GRPO / RLVR。

不要一开始安装 BIRD-RL 的 `bird_rl.yaml`。

## 7. 长任务运行规范

所有长任务必须满足：

- 使用 `nohup` 后台运行。
- stdout 和 stderr 写入 log 文件。
- log 文件路径记录清楚。
- 训练任务接入 SwanLab。
- 命令参数保存到脚本或配置文件中，避免只存在 shell history 里。

建议日志目录：

```text
logs/
  sft/
  grpo/
  eval/
```

建议输出目录：

```text
outputs/
  eval/
  predictions/
  reward_debug/
checkpoints/
```

## 8. 环境风险判断

低风险部分：

- 文档。
- 数据格式转换。
- SQLite 执行。
- reward 计算。
- evaluation。

中风险部分：

- ms-swift SFT。
- ms-swift GRPO。
- SwanLab 训练监控。

高风险部分：

- veRL。
- vLLM 和 CUDA 版本匹配。
- flash-attn 编译。
- Ray 多机多卡。
- 多轮 agent tool loop。

因此第一版环境策略是：先用最小依赖跑通 SQL RLVR 闭环，再逐步增加训练和加速组件。
