# SQL-RL 设计文档

## 1. 项目目标

本项目目标是做一个面向 post-training 实习展示的轻量 RLVR 项目：针对 Text-to-SQL 任务，构建从自然语言问题到 SQL 生成、SQLite 执行验证、reward 计算、GRPO 训练更新的完整闭环。

项目不是复现完整 BIRD-RL，而是借鉴 BIRD-RL 的 prompt、reward、evaluation 思路，做一个独立、可解释、可复现实验的 SQL RL 项目。

核心流程：

```text
Natural Language Question -> LLM generates SQL -> SQLite execution -> reward -> GRPO update
```

## 2. 项目主线

第一版只做单轮 SQL Generation + RLVR。

输入：

```text
question + database schema + optional evidence
```

输出：

```text
SQL
```

验证方式：

```text
执行模型 SQL 和 gold SQL，比较执行结果是否一致
```

训练方式：

```text
SFT 冷启动 -> GRPO/RLVR 训练 -> reward ablation -> evaluation
```

## 3. 暂不做的内容

第一阶段不做以下内容：

- 完整复现 BIRD-RL 四阶段训练。
- 多轮 SQL Agent。
- SQL Debugging 作为主线任务。
- veRL cluster / Ray 多机调度。
- 直接安装或复用 `bird_rl.yaml`。

这些内容可以作为后续增强，但不能阻塞第一版闭环。

## 4. 核心改进点

项目需要有一个区别于“直接跑 GRPO”的改进点。第一版推荐使用 cost-aware reward。

基础 reward：

```text
R_correct = 1 if execution result matches gold result else 0
```

cost-aware reward：

```text
R = R_correct
    - lambda_len * normalized_sql_length
    - lambda_invalid * invalid_sql
    - lambda_timeout * timeout
```

目标是在尽量保持执行正确率的同时，降低 SQL 长度、非法 SQL 比例和超时比例。

## 5. 实验设计

至少完成以下对比：

- Base model zero-shot / few-shot evaluation。
- SFT model evaluation。
- SFT + GRPO with correctness-only reward。
- SFT + GRPO with cost-aware reward。

主要指标：

- Execution Accuracy / EX。
- Invalid SQL rate。
- Timeout rate。
- 平均 SQL 长度。
- reward 曲线。
- 训练稳定性和样本效率。

## 6. 训练框架选择

第一版优先使用 ms-swift，因为使用者更熟悉，环境和训练脚本迁移成本较低。

TRL 作为备选方案，适合需要更透明地控制 GRPOTrainer 和 reward function 的情况。

veRL 暂不作为第一版训练框架。veRL 更适合后续多轮 agent、tool use、高并发 rollout、多机多卡训练工程展示。

训练框架不应绑定项目核心逻辑。数据处理、SQLite 执行器、reward、evaluation 应设计成框架无关模块，训练后端只负责调用它们。

## 7. BIRD-RL 的使用方式

BIRD-RL 不作为本项目的直接运行依赖，而作为参考项目使用。

主要参考：

- Prompt 构造思路。
- SQLite schema 提取方式。
- SQL execution reward 思路。
- BIRD evaluation 思路。
- 训练超参数的大致范围。

不复用：

- `bird_rl.yaml`。
- veRL patch。
- 多轮 agent tool loop。
- BIRD-RL 完整四阶段脚本。

## 8. 简历表达目标

项目最终应能表达为：

```text
Built a SQL RLVR training pipeline with execution-based verifiable rewards.
Implemented GRPO post-training on Text-to-SQL tasks and designed a cost-aware reward shaping method to balance execution accuracy and inference cost.
```

重点不是“调用了某个框架”，而是能讲清楚 RLVR 任务定义、reward 设计、GRPO 训练闭环、实验对比和失败样例分析。
